# 项目目标

## 项目概述

rdb-mhc (Resource Database - MetaHub & TemplateClient) 是一个面向**团队协作**的数据资产管理系统，使用 FastAPI + PostgreSQL + Svelte 技术栈构建。

**核心理念**：为散落的资源建立统一入口，通过多维度标签实现灵活分类，支持跨项目检索与交叉分析。

## 系统架构

```
┌─────────────────────────────────────────────────────┐
│                    rdb-mhc                          │
├──────────────────────┬──────────────────────────────┤
│      MetaHub         │     TemplateClient           │
│  ────────────────    │  ─────────────────────────   │
│  • 标签系统管理      │  • 数据存储与版本控制         │
│  • 层级化标签分类    │  • 多种数据类型的可视化       │
│  • 元数据管理        │  • 用户交互界面              │
│  • 资源元信息关联    │  • 数据的增删改查             │
└──────────────────────┴──────────────────────────────┘
```

## 核心功能

### 1. 标签系统 (MetaHub)

MetaHub 提供**多维度层级化标签系统**：

- **层级化标签**：支持任意深度的父子标签关系
- **多维度分类**：一个资源可从多个维度打标签
  - 项目维度：如「项目A」
  - 测试维度：如「测试批次-001」
  - 设备维度：如「设备型号-X」
  - 特性维度：如「产品特性-a/b/c」
- **层级搜索**：搜索父标签时自动包含子标签内容
- **多标签分类**：资源在同一分类下可有多个标签（非单一树状路径）

### 2. 数据资产管理 (TemplateClient)

TemplateClient 是数据的统一存储与展示层：

- **RDB 存储**：作为 Resource Database 存储各类轻量可序列化数据
- **多数据类型可视化**：针对已知数据类型，提供交互式可视化界面
  - 数值表格：可排序、筛选、导出
  - Mermaid 图表：渲染预览
  - 图片结构：网格/画廊视图
  - JSON 编辑器：树状/源码视图
- **资源模板**：不同类型数据有不同的快速添加界面，一键创建标准化资源
- **聚合分析**：支持同类数据的批量获取与趋势可视化
- **跨项目对比**：汇集同类型数据，支持跨项目数据对比

### 3. 搜索能力

- **标签搜索**：通过多维度标签组合筛选资源
- **关键字搜索**：全文关键字搜索，方便快速定位

### 4. 版本控制

- 记录资源变更历史
- 支持查看差异（diff）

### 5. API 接口

- 提供 RESTful API
- 支持分析程序推送结果到 RDB
- 可被其他应用复用 TC 的数据管理能力

### 6. 权限管理

- **基于角色的访问控制** (RBAC)
- 支持**私密资源**：可创建仅创建者可见的私有资源
- 团队成员可访问团队共享资源

## 典型使用场景

### 场景：硬件测试结果管理

**标签定义（MetaHub）：**
```
项目
├── SmartHome (父)
│   ├── SmartLight
│   └── SmartSensor
└── Wearable
    └── SmartWatch

设备型号
├── D-100
└── D-200

测试环境
├── 室内-常温
└── 户外-低温

特性
├── 功耗
├── 连接稳定性
└── 响应延迟
```

**数据存储（TemplateClient）：**
```
资源1: 测试记录-001
├── 类型: 性能数据表
├── 内容: { latency: 120ms, power: 2.3W, ... }
└── 标签: [SmartHome/SmartLight, D-100, 室内-常温, 响应延迟]

资源2: 测试记录-002  
├── 类型: Mermaid时序图
├── 内容: { mermaid: "sequenceDiagram..." }
└── 标签: [SmartHome/SmartSensor, D-200, 户外-低温, 连接稳定性]
```

**检索与可视化：**
- 搜索"室内-常温 + 响应延迟" → 找到所有相关测试，对比结果
- 聚合所有"功耗"数据 → 生成趋势图表
- 通过 API 推送新测试结果 → 自动关联标签

## 技术选型

- **后端**：FastAPI + asyncpg (Python)
- **数据库**：PostgreSQL
- **前端**：Svelte 5 + TypeScript

## 设计原则

- **统一入口**：所有资源集中管理，快速检索
- **多维度分类**：标签系统支持非树状的灵活分类
- **可复用**：TC 可作为其他应用的数据层，复用增删改查逻辑
- **易用性优先**：优先保证用户体验和操作便捷性
