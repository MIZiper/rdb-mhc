# MetaHub

## 简介

MetaHub的主要是定义标签，这些标签可以被下游使用附加至它们的数据中。
数据可以添加来自不同类别的标签，也可以添加一个类别下的多个标签。
添加了标签的数据后续可以通过标签进行筛选、聚合。

标签由不同的客户管理，每个客户又可以将标签分类，而在一个标签类别下，标签以父子树状关系存在。
如此当搜索父标签时，子标签数据也能够被搜索到。

MetaHub只负责标签的管理，以及在需要时提供搜索构建。

## 功能模块

MetaHub的代码定义在 mh-backend 和 mh-frontend 文件夹下。
前端使用 TypeScript+Svlete，后端使用 Python+PostgreSQL。

需要用户能够添加、修改标签、重新定义标签之间的关系。
因为标签需要被下游调用，用户还可以定义标签的可见性，来决定下游是否能够看到该标签。

除此之外，标签可以绑定至一个具体数据，表明该标签更详细的定义。
但是被绑定的数据定义在下游系统中，而其也可能附加着其它的标签。
MetaHub并不直接存储标签所绑定的数据，但是会记录标签被绑定的下游服务、该数据的链接与描述、该被绑定的数据所附加的标签。
用户可以在标签管理也为标签绑定其下游数据。

MetaHub主要有几个页面，用户的管理界面，进行分类与标签的属性管理；
列表显示被绑定的标签以及它所对应的数据信息。

注册客户的时候需要提供该客户所拥有下游应用的地址。
下游应用主要是TemplateClient的衍生应用，定义在 tc-backend 和 tc-frontend 中。

## 其它功能

在简单的标签搜索中，TemplateClient通过标签匹配来返回对应数据。
但是对于父标签搜索、复杂标签搜索，甚至结合被绑定数据的标签进行深度搜索聚合，MetaHub需要提供相关的算法知道下游应用返回哪些数据。
